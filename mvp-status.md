## Product MVP gap

This is not yet the terminal VS Code Product MVP.

The Product MVP requires:

1. `smith <ssh-host> <workspace>` opens a remote workspace over SSH.
2. Smith enters and restores terminal mode safely.
3. The user sees a VS Code-like workbench: navigation, Explorer, editor, panel, and status bar.
4. Explorer browses the remote workspace.
5. Editor opens, edits, undo/redos, saves, and detects conflicts for remote text files.
6. Search finds text in the remote workspace.
7. Integrated terminal runs a remote shell command.
8. Command palette exposes core commands.
9. Mouse click, wheel, resize, and keyboard-only operation work for the above.
10. End-to-end evidence is generated against an SSH target.
